# Milestone v1.9 Requirements Archive — Code Sandbox Hardening (Minimal)

**Shipped:** 2026-04-23 (code merged-ready; staging/prod apply + soak + rotations operator-gated)
**Goal:** Close the three known exfil paths from the Code action node (`/proc/<main>/environ`, K8s SA token, IRSA token) with the smallest possible infra + code change.

**Non-goals (explicit):** network-level egress policy, isolated-vm / native V8 isolates, cross-tenant primordial-pollution defence beyond what `node:vm` already provides, admission policies, Node permission flags, warm-pool / per-org budgets, sandbox audit log, tracing, public SLOs.

**Non-functional budget:** p99 Code-step wall-clock overhead ≤ 300 ms over current baseline.

---

## v1.9 Requirements (15)

### Sandbox Workload (WORK)

- [x] **WORK-01**: System runs user JavaScript in a separate Kubernetes Pod inside the existing `keeperhub` namespace, distinct from the main `keeperhub-{env}` Pod — a different Linux PID namespace than the main app so sandbox-escape code cannot `fs.readFileSync("/proc/<main-pid>/environ")` — **Outcome:** Manifests delivered (`deploy/keeperhub-sandbox/{staging,prod}/`). Staging/prod apply operator-gated. Live-verified on minikube via `deploy/keeperhub-sandbox/local/validation.yaml`.
- [x] **WORK-02**: Sandbox workload uses a dedicated ServiceAccount (`keeperhub-sandbox-{env}`) with zero RoleBindings/ClusterRoleBindings and no `eks.amazonaws.com/role-arn` IRSA annotation — **Outcome:** `sa.yaml` manifests. Live-verified on minikube.
- [x] **WORK-03**: Sandbox Pod sets `automountServiceAccountToken: false` at both the ServiceAccount object and the Pod spec; regression test asserts `/var/run/secrets/kubernetes.io/serviceaccount/token` does not exist inside the sandbox container — **Outcome:** `sa.yaml` + `pod-automount-patch.yaml`. Live-verified: token file returns ENOENT on minikube.

### Isolation Runtime (ISOL)

- [x] **ISOL-01**: Sandbox service evaluates user JavaScript via `node:vm.runInContext` inside a scrubbed child process, reusing the PR #953 approach — **Outcome:** `sandbox/src/run-code.ts` + `plugins/code/steps/run-code.ts` — byte-identical to commit `a93ce4b9`. CHILD_ENV_ALLOWLIST grep-asserted 6/6.
- [x] **ISOL-02**: Sandbox preserves existing JS surface and denials byte-for-byte — **Outcome:** 72 unit tests pass against both backends; captured console, bridged fetch, typed arrays, URL/Headers/Request/Response, AbortController, structuredClone, Intl, crypto.randomUUID, TextEncoder/Decoder, all Error subclasses; no require, no process, no setTimeout, no SharedArrayBuffer.
- [x] **ISOL-03**: Sandbox `fetch` blocks cloud-metadata hosts — **Outcome:** 5/5 BLOCKED_HOST_SUBSTRINGS (169.254.169.254, fd00:ec2::254, 169.254.170.2, metadata.google.internal, metadata.azure.com) grep-asserted in CHILD_SOURCE. Tests verify IMDS fetch rejection.

### Transport & Main-App Integration (INT)

- [x] **INT-01**: Main app has `lib/sandbox-client.ts` that calls the sandbox Pod over intra-cluster HTTP/1.1 with a persistent keep-alive `http.Agent` — **Outcome:** module-level `Agent({keepAlive:true, maxSockets:50})`. 5-sequential-request mock test confirms 1 TCP connection.
- [x] **INT-02**: Transport preserves PR #953 wire format — **Outcome:** `\u0001RESULT\u0002` sentinel byte-identical across `sandbox/src/run-code.ts` + `sandbox/src/index.ts` + `lib/sandbox-client.ts` + `plugins/code/steps/run-code.ts`. BigInt/Map round-trip tested.
- [x] **INT-03**: `plugins/code/steps/run-code.ts` selects the backend via `SANDBOX_BACKEND` env var — **Outcome:** module-init read, `?? "local"` default, `runRemote`/`runLocal` dispatcher.
- [x] **INT-04**: Existing unit test suite passes unchanged against BOTH `local` and `remote` backends — **Outcome:** CI `test-unit-sandbox-remote` job added. Live-verified on minikube: 72 pass against local, 72 pass against remote. Identical.

### Escape Regression Matrix (TEST)

- [x] **TEST-01**: `Error.constructor("return process")()` inside the sandbox returns a scrubbed process whose env contains no main-pod secret — **Outcome:** Live-verified on minikube with planted FAKE_CANARY in Pod env; only `["NODE_ENV","PATH"]` reach user code. Canary NOT leaked.
- [x] **TEST-02**: `fs.readFileSync("/proc/self/environ")` inside the sandbox contains no main-pod secret — **Outcome:** Live-verified on minikube; Node 24 blocks the fs escape path entirely via `process.mainModule` returning undefined (defence-in-depth). Where an older Node would have reached /proc/self/environ, the execve child env is CHILD_ENV_ALLOWLIST only so the canary would be scrubbed regardless.
- [x] **TEST-03**: `/proc/1/environ` and `/proc/<ppid>/environ` inside the sandbox do not return main-pod secrets — **Outcome:** Structurally held by separate-Pod topology (different PID namespace). Operator runs against staging/prod with planted main-pod sentinel.
- [x] **TEST-04**: `/var/run/secrets/kubernetes.io/serviceaccount/token` fails with ENOENT — **Outcome:** Live-verified on minikube. `automountServiceAccountToken: false` at SA + Pod level works as designed.
- [x] **TEST-05**: `/var/run/secrets/eks.amazonaws.com/serviceaccount/token` fails with ENOENT — **Outcome:** Live-verified on minikube. No IRSA annotation on SA means no AWS token is ever projected.

### Cleanup & Rollout (DEPLOY)

- [x] **DEPLOY-01**: Staging deploy lands the sandbox Pod + Service + scrubbed SA; `SANDBOX_BACKEND` flipped to `remote`; escape matrix passes in staging; 24h soak — **Outcome:** Artifacts + operator runbook delivered (`deploy/keeperhub-sandbox/staging/`, `deploy/keeperhub/staging/values.yaml`, `tests/e2e/sandbox-escape/`). Apply + soak operator-gated.
- [x] **DEPLOY-02**: Prod rollout mirrors staging; escape matrix re-run in prod; PR #953-style in-pod child_process path retired after 7 days stable — **Outcome:** Artifacts + runbooks delivered (`deploy/keeperhub-sandbox/prod/`, `deploy/keeperhub/prod/values.yaml`, `retirement-pr-scaffold.md`). Apply + soak + retirement PR operator-gated.

### Post-Deploy Credential Rotation (ROT)

- [x] **ROT-01**: After prod stable for 7 days, rotate the assume-breach high-value secrets one at a time — **Outcome:** `rotation-runbook.md` delivered with 11 secrets in documented order (AGENTIC_WALLET_HMAC_KMS_KEY first with 8-newest-version grace window, CDP_API_KEY_SECRET last), per-secret cycle, and KEEP-332 audit table template.
