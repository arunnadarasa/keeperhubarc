## Integration check

- Wire format continuity: `\u0001RESULT\u0002` sentinel byte-identical across `sandbox/src/run-code.ts`, `sandbox/src/index.ts`, `lib/sandbox-client.ts`, `plugins/code/steps/run-code.ts` — confirmed via grep and by the 72 unit tests passing against both backends.
- SANDBOX_BACKEND contract: main app reads from env at module init; staging values.yaml sets it to `remote`; prod values.yaml sets it to `remote`; CI `test-unit-sandbox-remote` job sets it to `remote`. All three converge on the same bit.
- SANDBOX_URL contract: default `http://localhost:8787` in dev, staging ClusterIP DNS in staging values, prod ClusterIP DNS in prod values. Three-envs, three-values, one URL variable.
- Image: sandbox/Dockerfile is the single source; built via docker-compose locally, via `minikube image load` for the live verification, published to ECR for staging/prod.
- E2E test contract: tests/e2e/sandbox-escape/escape-matrix.spec.ts uses `STAGING_URL`/`STAGING_API_TOKEN`/`EXPECTED_SENTINEL` env contract; operator wires these when running against staging/prod.
