## Milestone Goal

Close the three known exfil paths from the Code action node (`/proc/<main>/environ`, K8s SA token, IRSA token) with the smallest possible infra + code change.

## Non-Negotiable Cross-Cutting Decisions

1. Separate Kubernetes Pod in the same `keeperhub` namespace as the main app â€” different PID namespace via different Pod, same K8s ns, no new infra environment.
2. Keep `node:vm` + `child_process` approach from PR #953 (closed; code reused); `isolated-vm` / native V8 isolate explicitly out of scope for v1.9.
3. No NetworkPolicy in v1.9 â€” pod-level SA/IRSA removal closes the exploitable paths; deferred to v1.9.x if evidence requires.
4. PR #953 v8-serialized base64 body + `\x01RESULT\x02` sentinel wire format reused verbatim between main app and sandbox; preserves BigInt/Date/Map/Set round-trip required by v1.7 paid workflows.

## Key Decisions Logged

1. **PR #953 reuse over rewrite** â€” original PR closed but code reused verbatim at commit `a93ce4b9`; byte-for-byte fidelity enforced via grep-asserted acceptance criteria (CHILD_ENV_ALLOWLIST, BLOCKED_HOST_SUBSTRINGS, `\u0001RESULT\u0002` sentinel, outer kill timer at `timeoutMs + 1000`).
2. **`node:vm` over `isolated-vm`** â€” minimum-viable fix; `isolated-vm` deferred until evidence the pod boundary is insufficient.
