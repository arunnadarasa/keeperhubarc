**Frontmatter (opening lines of file):**
The file opens with a YAML frontmatter block. The `description` key value: `Run autonomous build-evaluate-iterate cycle for a single phase with optional code review gate`

**Heading:** `# /gsd:build-evaluate`

**Brief description:** 1-2 sentences explaining it orchestrates seed, dev server, builder, evaluator, and gap closure for a single phase without triggering the full execute-phase lifecycle.

**Usage section:**

```
/gsd:build-evaluate --phase N [--max-rounds N]
```
Arguments:
- `--phase N` — Required. Phase number to evaluate.
- `--max-rounds N` — Optional. Cap on gap-fix rounds for this run. Defaults to EVAL-CONFIG.yml `max_rounds` value. If provided and smaller than EVAL-CONFIG.yml value, the command's value wins (does NOT modify EVAL-CONFIG.yml).

**Step 1: Parse Arguments**

Parse `$ARGUMENTS` for `--phase N` and `--max-rounds N`. Error if `--phase` is missing. Set `PHASE_ARG` and `MAX_ROUNDS_ARG` (empty string if not provided).

**Step 2: Initialize**

```bash
INIT=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" init execute-phase "${PHASE_ARG}")
if [[ "$INIT" == @file:* ]]; then INIT=$(cat "${INIT#@file:}"); fi
```

Extract from INIT JSON: `phase_dir` (as PHASE_DIR), `phase_number` (as PHASE_NUMBER), `evaluator_model`, `executor_model`. Error if `phase_found` is false.

Detect EVAL-CONFIG.yml:
```bash
EVAL_CONFIG=$(ls "${PHASE_DIR}"/*-EVAL-CONFIG.yml 2>/dev/null | head -1)
```
If EVAL_CONFIG is empty: print "No EVAL-CONFIG.yml found in ${PHASE_DIR} -- nothing to evaluate." and exit.

Set effective max rounds:
```bash
if [ -n "${MAX_ROUNDS_ARG}" ]; then
  EFFECTIVE_MAX_ROUNDS="${MAX_ROUNDS_ARG}"
else
  CONFIG_MAX=$(grep "^max_rounds:" "${EVAL_CONFIG}" 2>/dev/null | cut -d: -f2 | tr -d ' ')
  EFFECTIVE_MAX_ROUNDS="${CONFIG_MAX:-3}"
fi
```

**Step 3: Code Review Gate (CMD-02)**

Check for an open PR before invoking /code-review:
```bash
PR_NUMBER=$(gh pr view --json number --jq '.number' 2>/dev/null)
```

If `PR_NUMBER` is empty or command exits non-zero: print "No open PR for current branch -- skipping code review gate" and proceed to Step 4.

If PR exists: invoke the `/code-review` skill (using `Skill(skill="code-review")` or SubTask call). If review returns blocking issues: present issues to user with AskUserQuestion offering options to (1) fix and retry or (2) continue anyway. The code-review gate is best-effort — do NOT permanently block evaluation if user chooses to continue.

**Step 4: Portless Detection**

Copy the portless detection block verbatim from execute-phase.md runtime_evaluation_gate:
```bash
PORTLESS_AVAILABLE="0"
PORTLESS_URL=""
if command -v portless >/dev/null 2>&1 && portless list 2>/dev/null | grep -q "keeperhub"; then
  PORTLESS_AVAILABLE="1"
  BRANCH=$(git rev-parse --abbrev-ref HEAD 2>/dev/null | tr '/' '-')
  if [ "${BRANCH}" = "main" ] || [ "${BRANCH}" = "staging" ]; then
    PORTLESS_URL="http://keeperhub.localhost:1355"
  else
    PORTLESS_URL="http://${BRANCH}.keeperhub.localhost:1355"
  fi
fi
```

**Step 5: Evaluation Round Loop**

Loop up to EFFECTIVE_MAX_ROUNDS times. Each iteration:

Spawn gsd-evaluator with `isolation="worktree"`:
```
Task(
  prompt="Evaluate phase ${PHASE_NUMBER} against EVAL-CONFIG.yml.
Phase directory: ${PHASE_DIR}
EVAL-CONFIG path: ${EVAL_CONFIG}
PORTLESS_AVAILABLE: ${PORTLESS_AVAILABLE}
PORTLESS_URL: ${PORTLESS_URL}
Run: detect portless (PORTLESS_AVAILABLE=1 means portless proxy is running), start dev server (use portless URL when PORTLESS_AVAILABLE=1, otherwise use configured port), seed data, evaluate each criterion, write EVAL.md.",
  subagent_type="gsd-evaluator",
  model="${evaluator_model}",
  isolation="worktree"
)
```

Read EVAL.md result:
```bash
LATEST_EVAL=$(ls -t "${PHASE_DIR}"/*-EVAL*.md 2>/dev/null | head -1)
EVAL_APPROVED=$(grep "^approved:" "${LATEST_EVAL}" 2>/dev/null | cut -d: -f2 | tr -d ' ')
EVAL_STATUS=$(grep "^status:" "${LATEST_EVAL}" 2>/dev/null | cut -d: -f2 | tr -d ' ')
EVAL_ROUND=$(grep "^round:" "${LATEST_EVAL}" 2>/dev/null | cut -d: -f2 | tr -d ' ')
```

Route based on result:
- `EVAL_APPROVED=true` (status: passed) -- proceed to Step 6 (success)
- `EVAL_STATUS=convergence_halt` -- proceed to Step 7 (escalation)
- `EVAL_APPROVED=false` AND `EVAL_ROUND >= EFFECTIVE_MAX_ROUNDS` -- proceed to Step 7 (escalation)
- `EVAL_APPROVED=false` AND `EVAL_ROUND < EFFECTIVE_MAX_ROUNDS` -- update STATE.md then spawn gap-fix executor (see below), then loop back

STATE.md update before gap-fix spawn:
```bash
FAILING_CRITERIA=$(grep "criterion:" "${LATEST_EVAL}" | cut -d: -f2 | tr -d ' "' | tr '\n' ',' | sed 's/,$//')
node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" state patch \
  --eval_round "${EVAL_ROUND}" \
  --eval_failing_criteria "${FAILING_CRITERIA}"
```

Gap-fix executor spawn with `isolation="worktree"`:
```
Task(
  prompt="Fix evaluation gaps for phase ${PHASE_NUMBER}.
EVAL.md path: ${LATEST_EVAL}
Read the gaps array from EVAL.md frontmatter. For each gap, implement the fix described in fix_hints.
Do not modify criteria that passed in previous rounds (criterion locking rule from EVAL.md).",
  subagent_type="gsd-executor",
  model="${executor_model}",
  isolation="worktree"
)
```
After executor completes, loop back to top of Step 5 (re-spawn gsd-evaluator for next round).

**Step 6: Success**

Print summary:
```
build-evaluate complete: Phase ${PHASE_NUMBER} approved.
Score: {score from EVAL.md}
EVAL report: ${LATEST_EVAL}
```

Do NOT update ROADMAP.md. Do NOT mark phase complete. Do NOT invoke gsd-verifier. This is a targeted evaluation tool, not a full phase lifecycle command.

**Step 7: Escalation**

Update STATE.md before presenting to user:
```bash
FAILING_CRITERIA=$(grep "criterion:" "${LATEST_EVAL}" | cut -d: -f2 | tr -d ' "' | tr '\n' ',' | sed 's/,$//')
node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" state patch \
  --eval_round "${EVAL_ROUND}" \
  --eval_failing_criteria "${FAILING_CRITERIA}"
```

Present options using AskUserQuestion:
```
## Evaluation Escalation Required

Phase ${PHASE_NUMBER} evaluation ${EVAL_STATUS == "convergence_halt" ? "reached convergence halt" : "exhausted max rounds (${EFFECTIVE_MAX_ROUNDS})"}.

Failing criteria: ${FAILING_CRITERIA}
EVAL.md: ${LATEST_EVAL}

This counts against the build-evaluate fix rounds counter (SAFE-02 fifth counter type).

Options:
1. Review EVAL.md manually and replan: /gsd:plan-phase ${PHASE_NUMBER} --eval-gaps
2. Abort evaluation -- treat this phase as complete without runtime verification
3. Increase max_rounds in EVAL-CONFIG.yml and retry: /gsd:build-evaluate --phase ${PHASE_NUMBER}
```

Do not proceed automatically. Wait for user input.
