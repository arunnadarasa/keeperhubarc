# Phase 29.2: Fix nonce DB conflict on sequential direct executions - Context

**Gathered:** 2026-03-29
**Status:** Ready for planning
**Source:** KEEP-198

<domain>
## Phase Boundary

`execute_check_and_execute` fails with a `pending_transactions` DB conflict when called after another write operation (`execute_transfer`) on the same wallet and chain. The condition evaluation (read) succeeds, but the action execution (write) fails with a nonce conflict.

### Root Cause 1: View functions go through write path

`app/api/execute/check-and-execute/route.ts` calls `writeContractCore()` for the action phase unconditionally, even when the action function is read-only (e.g., `decimals()`). This acquires a nonce session unnecessarily, creates a pending transaction record for a view call, and the transaction submission fails or conflicts.

### Root Cause 2: Nonce collision across direct executions

Each direct execution API call creates an independent nonce session via `withNonceSession()`. When two calls happen in quick succession, both get the same chain nonce because the first tx hasn't propagated yet. The upsert conflict on `pending_transactions(wallet, chain, nonce)` fails because both use the hardcoded execution ID `"direct-execution"`.

### Root Cause 3: Nonce session ignores DB pending transactions

`startSession()` uses `provider.getTransactionCount(address, "pending")` as source of truth, but if a tx was just submitted (milliseconds ago), the chain's pending count may not reflect it. The session should also check DB pending records and use `max(chainNonce, maxPendingDbNonce + 1)`.

</domain>

<decisions>
## Implementation Decisions

### From KEEP-198
- Route view/pure action functions in check-and-execute through readContractCore() instead of writeContractCore()
- Generate unique execution IDs for direct executions instead of hardcoded "direct-execution"
- Nonce session startSession() must account for unconfirmed pending transactions in DB
- Sequential direct execution calls on the same wallet must not produce DB conflicts
