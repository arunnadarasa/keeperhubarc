Critical settings that DIFFER from playwright.config.ts:
- `reuseExistingServer: false` -- MANDATORY for round isolation (each round starts a fresh server with fresh seed data)
- `retries: 0` -- failures in autonomous evaluation are real failures, not flakes
- `reporter: [["json", { outputFile: ".claude/eval-results.json" }]]` -- machine-readable output for scoring
- `globalSetup: "../../scripts/evaluate/seed-eval.ts"` -- relative from tests/e2e/playwright/ to scripts/evaluate/
- `webServer.command` uses port 3099 (not 3000 -- avoids collision with developer's running instance)
- `webServer.timeout: 120_000` -- pnpm dev includes discover-plugins which adds 10-20s cold start

portless integration: The webServer command checks `PORTLESS_AVAILABLE` env var. If set to "1", it wraps pnpm dev with portless for named URL assignment. Otherwise falls back to port 3099. The baseURL reads `PORTLESS_URL` first, falling back to `http://localhost:${evalPort}`.

Exact file to write:

```typescript
import { defineConfig, devices } from "@playwright/test";
import dotenv from "dotenv";
import { expand } from "dotenv-expand";

expand(dotenv.config());

const evalPort = parseInt(process.env.EVAL_PORT ?? "3099", 10);
const baseURL = process.env.PORTLESS_URL ?? `http://localhost:${evalPort}`;

const DEFAULT_DB_URL =
  "postgresql://postgres:postgres@localhost:5433/keeperhub";
const databaseUrl =
  process.env.DATABASE_URL && !process.env.DATABASE_URL.includes("${")
    ? process.env.DATABASE_URL
    : DEFAULT_DB_URL;
process.env.DATABASE_URL = databaseUrl;

const devServerCommand =
  process.env.PORTLESS_AVAILABLE === "1"
    ? `portless run --name keeperhub DATABASE_URL=${databaseUrl} pnpm dev`
    : `DATABASE_URL=${databaseUrl} pnpm dev --port ${evalPort}`;

export default defineConfig({
  globalSetup: "../../scripts/evaluate/seed-eval.ts",
  testDir: ".",
  testMatch: "**/*.test.ts",
  fullyParallel: false,
  forbidOnly: !!process.env.CI,
  retries: 0,
  workers: 1,
  reporter: [["json", { outputFile: ".claude/eval-results.json" }]],
  timeout: 60_000,
  use: {
    baseURL,
    trace: "off",
    screenshot: "only-on-failure",
    navigationTimeout: 60_000,
  },
  webServer: {
    command: devServerCommand,
    url: `http://localhost:${evalPort}`,
    reuseExistingServer: false,
    timeout: 120_000,
    stdout: "pipe",
    stderr: "pipe",
  },
  projects: [
    {
      name: "setup",
      testMatch: /auth\.setup\.ts/,
    },
    {
      name: "chromium",
      use: {
        ...devices["Desktop Chrome"],
        storageState: "tests/e2e/playwright/.auth/user.json",
      },
      dependencies: ["setup"],
    },
  ],
});
```

Important notes:
- `testDir: "."` since the config file lives inside tests/e2e/playwright/ -- tests are in the same directory
- Only two projects: `setup` and `chromium` (inviter and bystander projects are excluded -- evaluation uses the primary test user only)
- `storageState` path is relative to the repo root (same as playwright.config.ts)
- Do not add `globalTeardown` -- the webServer block handles server teardown automatically
- Do not add `TEST_API_KEY` extra header -- evaluation runs against local server where rate limiting is not relevant
